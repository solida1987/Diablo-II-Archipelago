TWO-FLAG ANTI-CHEAT DETECTION (F1 COLLECTION)
The Collection page detection rule rewritten with a strict two-flag
rule. A slot only marks "collected" when BOTH:
- Flag A: the item was observed on the ground with the engine's
  IFLAG_NEWITEM bit set (guaranteed-fresh from monster/chest/barrel
  /object — NOT from a player's drop or shared stash)
- Flag B: that same item is subsequently in your inventory

Items first observed in inventory (cube output, vendor purchase,
quest reward, items already in inv at character load, items
withdrawn from any stash including shared STK tabs) are locked
TAINTED on first observation and never auto-mark, even if you
later drop and pick them back up. Items dropped by another player
in MP also fail Flag A — by the time you see them on the ground,
the engine has already cleared IFLAG_NEWITEM.

The classification tracks each item by its per-game dwUnitId
(unique within a session). Sidecar version bumped from 1 to 2.
Existing characters get a one-time grace scan on first load post-
update so they don't lose pre-update collections; the sidecar is
then committed at v2 and strict mode applies for the rest of the
character's life.

================================================================
1.9.0 ADDITIONS — Reward redesign + bonus check categories
================================================================

REWARD CATALOG REDESIGN
Filler rewards are now individually-typed instead of generic placeholders.
Every quest reward in the spoiler file (and the AP spoiler) names exactly
what it gives:
  - Gold: random 1-10000 per location, pre-rolled at character creation
  - Experience: random 1-250000 per location
  - Trap variants: Monsters / Slow / Weaken / Poison (was just "Trap")
  - Boss Loot: specific boss (Andariel / Duriel / Mephisto / Diablo / Baal)
  - Drop: Random Charm / Set Item / Unique — pre-rolled specific item
    drops via QUESTS_CreateItem (items spawn unidentified, like a real
    monster drop). Spoiler shows the specific pre-rolled item name.

STANDALONE SPOILER FILE
New per-character spoiler at Game/Save/d2arch_spoiler_<char>.txt lists
every filler quest's pre-rolled reward. Re-written each character load
(deterministic per seed). Useful for offline runs and AP-mode preview.

BONUS CHECK CATEGORIES (opt-in, AP and standalone)
Six new toggleable check sources adding up to ~1494 extra AP locations:
  - Shrines: 50 per difficulty (escalating-chance roll on activation)
  - Urns/Jars: 100 per difficulty (smash to roll)
  - Barrels: 100 per difficulty (smash to roll)
  - Chests: 200 per difficulty (open to roll)
  - Set Piece Pickups: up to 127 first-time pickup checks
  - Gold Milestones: 17 lifetime-gold thresholds (10K -> 12M)

The chance starts at 10% on the first interaction for a slot and rises
+10% per consecutive miss, hitting a guaranteed 100% on the 10th try.
Resets to 10% after each hit. Average ~3.5 interactions per check.

All bonus locations are FILLER-ONLY — AP fill never places progression
items here, so unfilled slots cannot soft-lock the run. Toggle them via
title-screen buttons (column 2) or YAML options (default OFF).

PER-CHARACTER FILE LAYOUT
All per-character files now live in Game/Save/ next to the .d2s file
instead of being mixed into Game/Archipelago/. Migration is automatic
on first character load (legacy files are moved, not duplicated). The
shared Game/Archipelago/ folder now holds only account-wide files
(ap_settings.dat, ap_command.dat, shared_stash.dat, etc.).

F1 TRACKER UPDATES
Page 8 (Statistics) gained an "AP CHECK PROGRESS" section that shows
per-difficulty progress for shrines/urns/barrels/chests, the set-piece
pickup counter, and the gold-milestone progress including the next
threshold to reach. Visible only when at least one bonus category is
enabled.

ENTRANCE SHUFFLE NOW IN AP
The dead-end-cave entrance shuffle (1.9.0 System 1) was previously only
toggleable via d2arch.ini for standalone — it's now a proper YAML option
(`entrance_shuffle: bool`) that propagates through slot_data so multi-
world hosts can require it.
