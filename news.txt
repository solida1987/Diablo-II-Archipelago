=== IMPORTANT: CHARACTER SWITCHING ===
IF YOU WANT TO CREATE A NEW CHARACTER AFTER PLAYING ANOTHER ONE,
CLOSE THE GAME COMPLETELY AND RESTART IT BEFORE CREATING A NEW CHARACTER.
DO NOT SWITCH CHARACTERS FROM THE CHARACTER SELECT SCREEN.

=== IMPORTANT: INSTALLATION ===
The launcher requires you to point to your existing Diablo II + Lord of
Destruction installation. Original game files are copied from your own
install automatically. Only Diablo II Classic is supported (NOT Resurrected).

=== Beta 1.8.5 - New Features ===

INSTANT CONNECT-BUTTON FEEDBACK
Pressing Connect on the title screen now flips the button green the
moment the AP server authenticates, instead of waiting until you've
gone in-game and back out. All other settings buttons (Goal, XP,
Class filters, Quest toggles, Shuffle, Traps, Skill Hunting, Zone
Locking) now also auto-update to show the AP server's actual values
the instant the connection comes up — and stay locked until you
disconnect, so you can see at a glance what the server has
configured before clicking Single Player.

DEDICATED TREASURE-COW LOG
Every cow-spawn decision is recorded to Game/Archipelago/treasure_cow.log
with a wall-clock timestamp. Includes a 10-second heartbeat so you
can verify the spawn function is being called at all, plus per-decision
lines for every roll, skip-reason and spawn outcome. Helpful when
diagnosing "why didn't a cow show up here?" Same lines also mirror to
d2arch.log with a [TC] prefix.

AP GATE-KEY LOCATION HINTS (FOUNDATION)
The bridge now records the multiworld placement of every gate key
into ap_item_locations.dat. The DLL loads this file as the
foundation for an upcoming F4-tracker overlay that shows where each
gate key is hiding. Cross-world placements still need the !hint
mechanism — that completes in a follow-up.

=== Beta 1.8.4 - New Features ===

TRAPS TOGGLE
New "Traps" button on the title screen lets you opt out of trap fillers
entirely. When OFF, the filler pool redistributes that weight to gold,
stat points, skill points, reset points and boss-loot drops — pool
stays balanced.

SENDER + RECIPIENT ATTRIBUTION ON BANNERS
Multiworld banners now show who is involved in each transfer:
 - Incoming skills: "AP: Frozen Orb unlocked! (from test-d2-p2)" when
   another player found it for you.
 - Outgoing checks: "Den of Evil  ->  test-d2-p1" when the location
   belongs to someone else, or "Check: <quest>" for your own finds.
The Item Log on the right side of the AP panel matches the banner.

CRASH-FREE INJECTION
The DLL is loaded twice in the same process by the launcher path
(once directly, once through D2.Detours). Hook installation is now
guarded by named kernel mutexes and prologue checks, so the second
DLL image cannot smash the first one's trampolines and crash the
title screen.

=== Beta 1.8.0 - New Features ===

MULTI-TAB STASH
20 storage tabs per character (10 shared + 10 AP-only). Shift+right-click
items to auto-store or sell. Tabs persist in sidecar files; post-goal AP
characters get access to all 20 tabs.

SEMI-RANDOM ZONE LOCKING
New gated progression mode: each act has a fixed number of boss-kill gates
(4 for Acts 1/2/3/5, 2 for Act 4) with 3-4 randomized boss placements per
act baked into each character at creation. 18 gates per difficulty, up to
54 total in Full Hell mode. Kills send AP checks; gate keys unlock the
next region. 1024+ unique act-layouts per character.

SKILL LEVEL CAP RAISED 20 -> 99
Points invested past 20 are no longer silently capped. Skills can now
reach base level 99 with +skill gear stacking on top.

SIMPLIFIED GOAL + OPTIONS
Goal reduced from 15 options to 3: Full Normal, Full Nightmare, Full Hell.
AP options panel cleaned up to match the in-game settings 1:1 — removed
Game Mode (deprecated), Skill Pool Size, Starting Skills, 6 filler
weights, 15 act-preload overrides, Story Quests toggle, Shop Shuffle,
and I Play Assassin.

DEATHLINK UPGRADES
Incoming DeathLink traps now scale to your character level (no more
Colenzo spawning on a level-3 Sorc). Trap notification shows who died:
"DEATHLINK: <player> died - trap incoming!"

BUG REPORT / SUPPORT
Added Bug Form button in launcher alongside GitHub Issues link.
